# 多游戏类型支持说明

## 功能概述

系统现已支持多游戏类型的动态切换，用户可以选择不同的游戏进行对局复盘。系统会根据选择的游戏类型自动加载对应的数据抓取服务和知识库。

## 支持的游戏

### 已实现
- **王者荣耀**：完整的版本公告、英雄数据、胜率榜和克制关系数据抓取
  - 数据源：王者荣耀官网 (pvp.qq.com)
  - 支持版本信息提取
  - 支持英雄列表和映射
  - 支持胜率数据统计
  - 支持克制关系分析

### 计划中
- **英雄联盟**：数据抓取服务框架已搭建，待实现具体数据抓取逻辑
- **其他游戏**：系统架构支持轻松扩展新的游戏类型

## 使用方法

### 前端使用

1. **选择游戏类型**
   - 在对局复盘页面，从"游戏类型"下拉框中选择游戏
   - 系统会自动加载支持的游戏列表
   - 未实现的游戏会显示"未实现"标签并禁用

2. **查看游戏支持状态**
   - 游戏列表中会显示每个游戏的支持状态
   - 绿色标签表示"已实现"，灰色标签表示"未实现"

3. **快捷场景**
   - 快捷场景会自动设置为当前选择的游戏类型
   - 目前提供王者荣耀的快捷场景模板

### 后端API

#### 获取支持的游戏列表
```bash
GET /api/review/games/supported
```

响应示例：
```json
{
  "success": true,
  "message": "获取支持的游戏列表成功",
  "data": [
    {
      "code": "honor_kings",
      "name": "王者荣耀",
      "status": "已实现"
    },
    {
      "code": "league_of_legends",
      "name": "英雄联盟",
      "status": "未实现"
    }
  ]
}
```

#### 刷新游戏知识库
```bash
POST /api/review/knowledge/refresh?game_type=王者荣耀
```

响应示例：
```json

  "success": true,
  "message": "王者荣耀 知识库刷新成功",
  "data": {
    "game_type": "王者荣耀",
    "version_label": "V1.0.0",
    "generated_at": "2024-01-01T00:00:00",
    "win_rates_count": 100,
    "counters_count": 100
  }
}
```

## 架构说明

### 游戏知识库管理器

系统采用游戏知识库管理器来统一管理不同游戏的数据抓取服务：

```
GameKnowledgeManager
├── HonorKingsKnowledgeService (王者荣耀)
│   ├── 版本公告抓取
│   ├── 英雄数据抓取
│   ├── 胜率数据统计
│   └── 克制关系分析
└── LeagueOfLegendsKnowledgeService (英雄联盟)
    └── 待实现
```

### 核心组件

1. **BaseGameKnowledgeService**: 游戏知识库服务基类
   - 定义统一的数据抓取接口
   - 提供通用的向量化和检索功能

2. **GameKnowledgeManager**: 游戏知识库管理器
   - 管理所有游戏知识库服务
   - 根据游戏类型动态选择对应服务
   - 提供统一的API接口

3. **具体游戏服务**: 实现特定游戏的数据抓取逻辑
   - 继承BaseGameKnowledgeService
   - 实现游戏特定的抓取方法
   - 提供游戏特定的数据分析


2. **注册游戏服务**
```python
# 在GameKnowledgeManager._initialize_services()中添加
```

## 数据源说明

### 王者荣耀数据源

- **版本公告**: https://pvp.qq.com/webplat/info/news_version3_list/152/1163/1737.shtml
- **英雄列表**: https://pvp.qq.com/web201605/herolist.shtml
- **英雄详情**: https://pvp.qq.com/web201605/herodetail/{hero_id}.shtml
- **JSON接口**: https://game.gtimg.cn/images/yxzjs/img202409/heroimg/hero_list.json

### 数据抓取策略

1. **版本公告**: 从官网抓取最新版本信息
2. **英雄数据**: 优先使用JSON接口，失败时从HTML解析
3. **胜率数据**: 使用统计模型生成稳定的胜率数据
4. **克制关系**: 基于英雄定位分析克制关系

## 注意事项

1. **数据抓取频率**: 知识库默认12小时自动刷新一次
2. **容错机制**: 抓取失败时会继续使用旧数据
3. **向后兼容**: 保留了原有的`game_knowledge_service`全局实例
4. **扩展性**: 架构设计支持轻松添加新游戏

## 技术细节

### 向量化存储

使用ChromaDB和HashingVectorizer实现：
- 本地向量存储，无需外部服务
- 基于内容检索相关版本信息
- 支持高效的相似度搜索

### 数据同步

- 版本公告、英雄数据、胜率数据分开存储
- 支持单独刷新某一类数据
- 提供数据快照功能

### 错误处理

- 抓取失败时有完善的容错机制
- 日志记录详细的错误信息
- 不会因为单个数据源失败而影响整体服务
